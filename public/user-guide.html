📚 Panduan Pengguna Aplikasi FAD

Fixed Asset Disposal - Sistem Manajemen Aset Tetap

📋 Tentang Aplikasi

FAD (Fixed Asset Disposal) adalah sistem untuk mengelola proses pembuangan atau pengalihan aset tetap perusahaan.

TPS (Tempat Penyimpanan Sementara) adalah lokasi penyimpanan sementara untuk aset-aset yang sedang dalam proses disposal sebelum dipindahkan atau dibuang secara permanen.

🔐 Login & Autentikasi

Cara Login

  1. Masukkan Username dan Password
  2. Klik tombol Login
  3. Token akan berlaku selama 1 hari
  4. Setelah login, sistem akan mengarahkan ke dashboard sesuai dengan role user

Role User

  • EXTERNAL Hanya akses Dashboard FAD
  • INTERNAL Akses Dashboard FAD dan TPS dengan pilihan dashboard
  • ADMIN Akses penuh termasuk Admin Panel

👥 Dashboard Berdasarkan Role

User EXTERNAL

  • Langsung diarahkan ke Dashboard FAD
  • Tidak bisa mengakses Dashboard TPS atau Admin Panel

User INTERNAL

  • Setelah login, muncul Dashboard Selector
  • Dapat memilih antara Dashboard FAD atau Dashboard TPS
  • Dapat berpindah dashboard menggunakan switcher di pojok kanan atas

User ADMIN

  • Langsung diarahkan ke Admin Panel
  • Dapat mengakses semua dashboard melalui Admin Panel

🌲 Dashboard FAD

Fitur Utama

📊 Cards Overview

  • Total Data
  • Open Status terbuka
  • Progress Status dalam proses
  • Close Status selesai
  • Hold Status ditahan

⚡ Quick Actions

  • Add Data: Tambah data FAD baru
  • User Guide: Buka panduan pengguna
  • Import: Import data dari file

🔍 Filter & Search

  • Filter berdasarkan Area
  • Filter berdasarkan Status (Open/Progress/Close/Hold)
  • Search berdasarkan No FAD

Status FAD

  • Open Data baru yang belum diproses
  • Progress Data sedang dalam proses
  • Close Data sudah selesai diproses
  • Hold Data ditahan karena suatu alasan

🏗️ Dashboard TPS

Fitur TPS (Tempat Penyimpanan Sementara)

  • Total TPS
  • Status tracking
  • Data summary
  • Tambah TPS baru
  • Edit informasi TPS

📝 Manajemen Data

Tambah Data Baru

  1. Klik tombol Add Data
  2. Isi form dengan informasi:
    • No FAD
    • Area
    • Tanggal Terima FAD
    • Tanggal Terima BBM
    • Tanggal Serah Terima
    • Status awal
    • Keterangan (opsional)
  3. Klik Save untuk menyimpan

⚠️ Edit Data

  1. Klik ikon Edit pada data yang ingin diedit
  2. Form akan terpopulasi dengan data existing
  3. Penting: Tanggal akan otomatis terisi sesuai data yang ada
  4. Ubah informasi yang diperlukan
  5. Klik Update untuk menyimpan perubahan

📊 Status Tracking

Mengubah Status

  1. Pada data table, klik dropdown status
  2. Pilih status baru:
    • Open → Progress → Close
    • Atau set ke Hold jika ada masalah
  3. Tambah keterangan jika diperlukan
  4. Status akan otomatis terupdate

Hold Status

  • Data dengan status Hold dapat dilihat di halaman khusus
  • Akses melalui card "Hold" di dashboard
  • Menampilkan semua data yang statusnya Hold
  • Dapat diubah kembali ke status normal

📸 Upload Foto

Cara Upload Foto

  1. Pada detail data, klik Upload Photo
  2. Pilih file foto dari komputer
  3. Foto akan otomatis di-resize untuk thumbnail
  4. Foto dapat dilihat dalam gallery view
  5. Support format: JPG, PNG

⚙️ Admin Panel

Fitur Admin (Khusus ADMIN Role)

  • User Management: Tambah, edit, set role user
  • System Management: Logs, backup, configuration
  • Dashboard Access: Akses ke semua dashboard

🛡️ Security Logs (Admin)

⚠️ Akses Terbatas: Fitur ini hanya tersedia untuk user dengan role ADMIN

Cara Mengakses Security Logs

  1. Login sebagai user dengan role ADMIN
  2. Buka Admin Dashboard (FAD)
  3. Klik tombol Main Menu (ikon tiga garis)
  4. Pilih Security Logs dari dropdown menu
  5. Modal Security Logs akan terbuka

Fitur Security Logs

📊 Statistics Dashboard

  • Total Events: Jumlah total log security
  • Failed Logins: Jumlah percobaan login gagal
  • Critical Events: Event dengan severity CRITICAL
  • Unique IPs: Jumlah IP unik yang tercatat

🔍 Filter & Search

  • Search: Cari berdasarkan IP, user agent, atau details
  • Event Type: LOGIN_FAILURE, UNAUTHORIZED_ACCESS, SUSPICIOUS_ACTIVITY
  • Severity Level: INFO, WARN, ERROR, CRITICAL
  • Date Range: Filter berdasarkan rentang tanggal
  • Real-time Updates: Log otomatis terupdate

Tipe Security Events

  • Login Failures: Percobaan login yang gagal
  • Unauthorized Access: Akses tanpa autorisasi ke endpoint
  • Suspicious Activity: Multiple failed attempts atau pola aneh
  • Token Issues: JWT token expired atau invalid
  • Input Validation: Upaya input yang tidak valid/berbahaya

🚨 Monitoring Penting

  • Periksa security logs setidaknya sekali sehari
  • Perhatikan pola suspicious activity
  • Monitor IP addresses dengan multiple failures
  • Watch for unusual access patterns

Detail Security Event Examples

🔴 Login Failures

Type: LOGIN_FAILURE
Severity: WARN
Details: Invalid credentials for user: admin@example.com
IP: 192.168.1.100

🚫 Unauthorized Access

Type: UNAUTHORIZED_ACCESS
Severity: ERROR
Details: Access denied to /admin/users - insufficient permissions
User: user_123

⚠️ Suspicious Activity

Type: SUSPICIOUS_ACTIVITY
Severity: CRITICAL
Details: Multiple failed login attempts detected
IP: 192.168.1.50
Count: 5 attempts in 2 minutes

🔑 Token Issues

Type: TOKEN_ERROR
Severity: ERROR
Details: JWT token expired or invalid
IP: 192.168.1.80

File & Configuration Details

📁 Backend Files Location

  • FAD-BE/controllers/logController.js - Log API controller
  • FAD-BE/routes/logRoutes.js - Log API routes
  • FAD-BE/middlewares/securityLogger.js - Security logging utility
  • FAD-BE/middlewares/validation.js - Input validation with logging
  • FAD-BE/logs/security.log - Security events log file

🎨 Frontend Files Location

  • FAD-FE/src/components/SecurityLogs.vue - Security logs interface
  • FAD-FE/src/views/Admin.vue - Admin dashboard integration

⚙️ Environment Configuration

# Backend (.env file)
LOG_LEVEL=info                   # Minimum log level
LOG_RETENTION_DAYS=30           # Days to keep logs
MAX_LOGIN_ATTEMPTS=5            # Max failed attempts before suspicious
RATE_LIMIT_WINDOW=15            # Rate limit window in minutes
RATE_LIMIT_MAX=100              # Max requests per window

🔄 Log Management

  • Log Rotation: Files dirotasi otomatis setiap hari
  • Auto Cleanup: Log lama dibersihkan setelah 30 hari
  • File Size Limit: Maximum 50MB per file
  • Backup: Regular backup untuk disaster recovery

🧪 Testing & Development Guide

⚠️ Developer Section: Bagian ini ditujukan untuk developer dan QA team untuk testing sistem changelog dan security logging.

Backend Testing

🚀 Start Backend Server

cd FAD-BE
npm start

Server harus berjalan tanpa error di http://localhost:5001

📡 Test API Endpoints (Postman/curl)

Get Changelog Logs:

GET /api/changelog/logs
Headers: Authorization: Bearer <admin_token>

Get Changelog Stats:

GET /api/changelog/stats
Headers: Authorization: Bearer <admin_token>

Export Changelog:

GET /api/changelog/export
Headers: Authorization: Bearer <admin_token>

Get Security Logs:

GET /api/logs/security
Headers: Authorization: Bearer <admin_token>

Frontend Testing

🎨 Start Frontend Server

cd FAD-FE
npm run dev

🔍 Access Testing Interface

  1. Login sebagai ADMIN
  2. Navigate ke Admin Dashboard (FAD)
  3. Click Main Menu (≡) → Change Logs/Security Logs
  4. Modal viewer akan terbuka

End-to-End Testing Scenarios

📊 Scenario 1: Generate Test Changelog Data

  • Create FAD Entry → Should log "FAD CREATE"
  • Update FAD Entry → Should log "FAD UPDATE"
  • Delete FAD Entry → Should log "FAD DELETE"
  • Upload Photo → Should log "PHOTO UPLOAD"
  • Delete Photo → Should log "PHOTO DELETE"

🔐 Scenario 2: Generate Security Log Data

  • Failed Login → Should log "LOGIN_FAILURE"
  • Access Denied → Should log "UNAUTHORIZED_ACCESS"
  • Multiple Failed Attempts → Should log "SUSPICIOUS_ACTIVITY"
  • Invalid Token → Should log "TOKEN_ERROR"
  • Invalid Input → Should log "VALIDATION_ERROR"

📱 Scenario 3: Test Viewer Interface

  1. Open Change Logs/Security Logs dari Admin Menu
  2. Verify Statistics Dashboard shows correct counts
  3. Test Filtering:
    • Filter by Model/Event Type (FAD, USER, PHOTO, etc)
    • Filter by Action/Severity (CREATE, UPDATE, DELETE, etc)
    • Search functionality
    • Date range filtering
  4. Test Pagination (jika ada banyak entries)
  5. Test Detail View (click "View Details")
  6. Test Export CSV functionality

💡 Tips & Troubleshooting

✅ Update Log Terbaru (September 2025)

  • 🆕 Security Logs: Monitoring keamanan untuk ADMIN
  • 🆕 Change Logs: Riwayat perubahan data untuk ADMIN
  • ✅ Dashboard Selector: User INTERNAL dapat memilih dashboard
  • ✅ Dashboard Switcher: Navigasi antar dashboard
  • ✅ Hold Status Page: Halaman khusus untuk data Hold
  • ✅ Role-based Redirect: Auto redirect sesuai role setelah login
  • ✅ Date Field Fix: Tanggal otomatis terisi saat edit data
  • ✅ User Guide Local: Guide sekarang tersedia offline

Tips Penggunaan

  • Tanggal: Selalu isi tanggal dengan format DD/MM/YYYY
  • Status: Gunakan status Hold untuk data yang bermasalah
  • Navigation: User INTERNAL dapat berpindah dashboard dengan switcher
  • Search: Gunakan No FAD untuk pencarian cepat

Troubleshooting

  • Login Gagal: Periksa username/password, atau hubungi admin
  • Data Tidak Muncul: Periksa filter yang aktif
  • Upload Gagal: Periksa ukuran file dan format
  • Tanggal Tidak Tersimpan: Pastikan format tanggal benar
  • Dashboard Tidak Muncul: Periksa role user Anda
  • Menu Security/Change Logs Tidak Muncul: Pastikan login sebagai ADMIN
  • Security Logs Kosong: Generate aktivitas terlebih dahulu (login/logout)
  • Change Logs Tidak Terupdate: Refresh halaman atau lakukan operasi CRUD
  • Export Gagal: Pastikan browser mengizinkan download file
  • Filter Tidak Berfungsi: Klik tombol "Apply Filters" setelah mengubah kriteria

Keyboard Shortcuts

  • Ctrl + F: Quick search
  • Enter: Submit form
  • Esc: Cancel/Close modal
